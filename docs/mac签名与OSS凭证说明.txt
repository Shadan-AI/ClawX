ClawX 发布常见问题：macOS 签名 与 阿里云 OSS 凭证
================================================================================
（说明文档，非代码；按需修改路径或仓库名即可。）

--------------------------------------------------------------------------------
一、Apple / macOS 代码签名与公证
--------------------------------------------------------------------------------

现象：
  - 日志中出现 identity=null、skipped macOS code signing
  - arm64 requires signing, but identity is set to null 等提示

含义：
  - 当前构建被配置为「未签名」或证书未注入，仍会产出 zip/dmg，但用户侧
    Gatekeeper 体验差；面向公众分发时，签名 + 公证（Notarization）很重要。

代码是否要改：
  - 一般不需要为「能签名」改业务代码。electron-builder.yml 已包含 mac 相关
    配置；CI 在 .github/workflows/release.yml 里根据 Secrets 分支：
    - 证书 + 公证信息齐全 → pnpm run package:mac（签名 + 公证）
    - 否则 → package-mac.mjs 带 --config.mac.identity=null --config.mac.notarize=false

你需要在 GitHub 仓库配置（Settings → Secrets and variables → Actions）：
  - MAC_CERTS              开发者证书（如 .p12，按团队文档格式提供）
  - MAC_CERTS_PASSWORD     证书密码
  - APPLE_ID               Apple ID（用于公证）
  - APPLE_APP_SPECIFIC_PASSWORD   App 专用密码
  - APPLE_TEAM_ID          团队 ID

配齐后 Release 工作流中的 mac 构建会自动走签名路径；本地打包若也要签名，需自行
导出 CSC_LINK / CSC_KEY_PASSWORD 及公证相关环境变量（与 electron-builder 文档一致）。

--------------------------------------------------------------------------------
二、阿里云 OSS：accessKeyID and ecsUrl are both empty
--------------------------------------------------------------------------------

现象：
  - Run CHANNEL="latest" 后出现 Error: accessKeyID and ecsUrl are both empty
  - 进程以 exit code 1 结束

说明：
  - CHANNEL="latest" 通常只是版本解析结果（无 -alpha/-beta 等预发布段时为 latest），
    不是错误原因。
  - 失败来自 upload-oss 任务：ossutil 未读到有效的 AccessKey，等价于凭证为空。

你需要在 GitHub 仓库 Secrets 中配置（与 release.yml 中引用一致）：
  - OSS_ACCESS_KEY_ID       阿里云 RAM 用户 AccessKey ID
  - OSS_ACCESS_KEY_SECRET   对应 Secret

上述密钥需对仓库工作流中使用的 OSS Bucket（如 valuecell-clawx）及目标路径有上传权限。

若暂时不使用 OSS 上传：
  - 不配密钥则 upload-oss 会持续失败；需要么补全 Secrets，要么改工作流在未配置时
    跳过 OSS（属 CI 策略变更，与「仅补凭证」不同）。

--------------------------------------------------------------------------------
三、对照简表
--------------------------------------------------------------------------------

问题类型              处理方式
--------------------------------------------------------------------------------
mac 未签名 / 公证缺失  在 GitHub 配置 Apple + 证书相关 Secrets；一般不改业务代码
OSS 上传失败          在 GitHub 配置 OSS_ACCESS_KEY_ID / OSS_ACCESS_KEY_SECRET

## 3. 动画效果

### 输入框展开/收起动画

```css
transition-all duration-300
```

**变化过程（300ms）：**
1. 容器宽度：768px → 672px（平滑过渡）
2. 内边距：16px/24px → 8px（平滑过渡）
3. 输入框内边距：8px → 4px（平滑过渡）
4. 附件预览：opacity 1 → 0，height auto → 0（淡出 + 收起）

## 5. 触发条件

### 展开触发（满足任一）

```
isInputExpanded = isAtBottom || isInputFocused

条件 1: isAtBottom = true
  - 滚动位置距离底部 < 100px
  
条件 2: isInputFocused = true
  - 用户点击输入框
  - 输入框获得焦点
```

### 收起触发（同时满足）

```
isInputExpanded = false

条件 1: isAtBottom = false
  - 滚动位置距离底部 >= 100px
  
条件 2: isInputFocused = false
  - 输入框失去焦点
```

## 8. 常见问题

### Q: 我看不到任何变化
**A:** 确保：
1. 页面有足够内容可以滚动（发送 10+ 条消息）
2. 向上滚动超过 100px
3. 打开控制台查看日志
4. 检查浏览器缓存（Ctrl+Shift+R 强制刷新）

### Q: 输入框一直是展开的
**A:** 检查控制台日志：
- 如果 `isAtBottom: true`，说明你在底部，这是正常的
- 尝试向上滚动，`isAtBottom` 应该变为 `false`

### Q: 滚动时没有反应
**A:** 检查：
1. 控制台是否有 `[Chat] Scroll:` 日志
2. 如果没有，可能是滚动容器问题
3. 尝试刷新页面

### Q: Gateway 状态在哪里？
**A:** 位置：
- 侧边栏底部
- 设置按钮上方
- 如果侧边栏收起，只显示圆点

### Q: 视觉变化太小
**A:** 主要变化：
- 宽度：768px → 672px（-96px）
- 高度：自动 → 44px（固定）
- 内边距减少
- 附件和 Target 标签隐藏

如果还是看不清楚，可以：
1. 添加附件（会更明显）
2. 选择 Target Agent（会更明显）
3. 输入多行文字（高度变化更明显）
